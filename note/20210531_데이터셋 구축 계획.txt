데이터 셋 구축 계획

1. 의무기록사본을 사진으로 2000장을 찍는다.(각 10장씩)
2. 사진을 모두 로컬& 구글드라이브에 저장한다.
3-1. 사진 한장을 파이썬으로 코드를 작성해 detection을 완성시킨다.
4. 이후 labelme를 이용해 라벨링한다.
장점: ocr을할때 dectection이 되어있기때문에 모델을 이용하여 좌표및 성능 완성도가 높을 수 있다.
단점: 데이터셋 구축이 목표인데 detection까지 신경써야하기때문에 진행 속도가 느리다.

3-2. 파이썬 없이 라벨미로 노가다성 라벨링을 완성시킨다.
4. json 형식의 데이터셋을 완성시킨다.
5. 파이썬에서 json 형식의 데이터셋을 mnist형식처럼 데이터셋을 재 구성한다.
- 28*28로 모두를 어떻게 맞출수 있을까? 사이즈 변환? 여백 추가?
- 동물을 판별하는 데이터 셋은 사이즈가 각각 완전 다를걸로 예상된다. 어떤 구조인지 살펴보자.
6. 모델을 생성한다.
7. 테스트를 실행한다.
장점: 빠른 테스트로 모델을 만들 수 있는지 없는지 알 수 있다.(단, 라벨미로 수천장 만들지 말고 1장으로만 테스트할경우)
단점: detection 알고리즘이 없기때문에 차후에 개발이 필요하다.


결론: 3-2의 방식 선택.
1. 빠른 테스트 이후 디텍션코드를 구현하고 데이터셋을 집중해서 만든다. 
2. 단, 데이터 셋을 만들면서 디텍션이 잘 되는지 사진마다 테스트가 필요하다.
3. 데이터셋 만드는 방법, ocr 논문, opencv교재 등은 수시로 보며 기본을 익혀야한다.
